# Спецификация: поддержка горячих клавиш в Auto-Wall

## Цель
Добавить пользовательские горячие клавиши для основных режимов работы с масками и режимов редактирования стен в интерфейсе Auto-Wall.

## Область изменений
- GUI в `src/gui/app.py`
- Панель редактирования стен в `src/gui/export_panel.py`
- Обработка ввода в `src/gui/image_viewer.py`
- Возможные тесты в `tests/`

## Новые возможности
1. Горячие клавиши для режима рисования маски
   - `D` — выбрать режим Draw
   - `E` — выбрать режим Erase
2. Горячие клавиши для выбора инструментов рисования
   - `1` — Brush
   - `2` — Line
   - `3` — Rectangle
   - `4` — Circle
   - `5` — Ellipse
3. Горячие клавиши для режима работы со стенами
   - `W` — Draw Walls
   - `I` — Edit Mode
   - `X` — Delete Mode
   - `P` — Draw Doors / Portal Mode
4. Универсальные хоткеи
   - `Esc` — отмена текущей операции (прекращение рисования, сброс временного выделения)
   - `Delete` — удалить выбранный фрагмент стены / портал в режиме редактирования
   - `Ctrl+Z` — Undo (уже реализовано)

## Поведенческая логика
- Клавиши должны работать независимо от того, активен ли фокус на основном окне или на области отображения изображения.
- Переключение режимов должно синхронизировать состояние переключателей и текущие флаги (`uvtt_draw_mode`, `uvtt_edit_mode`, `uvtt_delete_mode`, `uvtt_portal_mode`, `draw_radio`, `erase_radio` и т.д.).
- При отмене текущей операции должны сбрасываться временные состояния рисования:
  - `drawing_new_wall`
  - `drawing_new_portal`
  - `new_wall_start`
  - `new_wall_end`
  - `new_portal_start`
  - `new_portal_end`
- При удалении выбранной стены должен срабатывать существующий механизм удаления из панели стен.
